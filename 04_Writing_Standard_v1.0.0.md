# 1. Purpose

This document defines the writing standards for every canonical document
in the AISOS repository. Its objective is to ensure that documentation
remains consistent, professional, reusable, and understandable by both
humans and AI systems.

# 2. Writing Philosophy

## Standard

Write as a professional studio documenting a long-term production.

## Principles

1.  Clarity over cleverness.
2.  Consistency over novelty.
3.  Precision over ambiguity.
4.  Reusability over convenience.
5.  Canon before creativity.

## Rationale

Documentation should reduce uncertainty, preserve institutional
knowledge, and enable collaboration across years of production.

------------------------------------------------------------------------

# 3. Tone and Voice

## Tone

-   Professional
-   Confident
-   Neutral
-   Cinematic when appropriate
-   Free of marketing language

## Voice

-   Active voice by default.
-   Use imperative language for standards ("Use...", "Avoid...",
    "Document...").
-   Separate facts, assumptions, and recommendations.

------------------------------------------------------------------------

# 4. Writing Rules

-   One primary topic per document.
-   One idea per paragraph where practical.
-   Prefer bullet lists for procedures.
-   Use numbered lists for ordered steps.
-   Expand acronyms on first use.
-   Avoid unexplained jargon.

------------------------------------------------------------------------

# 5. AI Collaboration Rules

Every document should:

-   State assumptions explicitly.
-   Avoid inventing canon.
-   Flag missing information.
-   Reference document IDs instead of conversation history.
-   Be compatible with multiple AI systems.

------------------------------------------------------------------------

# 6. Examples

## Good

> "Update CHAR-004 after approval of RFC-012."

Reason: References permanent identifiers.

## Poor

> "Update the character document we discussed earlier."

Reason: Depends on chat history and is ambiguous.

------------------------------------------------------------------------

# 7. Anti-Patterns

Avoid:

-   Vague language.
-   Repeating information across documents.
-   Mixing decisions with brainstorming.
-   Contradicting established canon.
-   Emotional or promotional wording in standards documents.

------------------------------------------------------------------------

# 8. Quality Checklist

-   [ ] Purpose is clear.
-   [ ] Tone is consistent.
-   [ ] Terminology follows canon.
-   [ ] References use document IDs.
-   [ ] Grammar reviewed.
-   [ ] Markdown validated.
